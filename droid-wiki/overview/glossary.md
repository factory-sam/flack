# Glossary

Project-specific terms and vocabulary used throughout Flack and this wiki.

| Term                                                          | Meaning                                                                                                                                                                                             |
| ------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Organization (org)**                                        | The tenant boundary. Every row carries an `org_id`; RLS isolates data per organization. Created at signup.                                                                                          |
| **Profile**                                                   | A user record in `public.profiles`, keyed to an `auth.users` id, scoped to one org, with a `role` of `admin` or `member`.                                                                           |
| **Channel**                                                   | A conversation container. `channels.type` is `public`, `private`, or `dm`.                                                                                                                          |
| **DM**                                                        | A direct-message channel (`type = 'dm'`), named by joining the two user ids with `:`.                                                                                                               |
| **Channel member**                                            | A row in `channel_members` linking a profile to a channel with a `role` (`owner` or `member`) and `last_read_at`.                                                                                   |
| **Thread**                                                    | A reply chain. A reply sets `messages.parent_id` to the root message's id.                                                                                                                          |
| **Optimistic message**                                        | A message rendered locally with `pending: true` before the server confirms it; reconciled (or marked `failed`) afterward. See [Realtime and optimistic UI](../features/realtime-and-optimistic.md). |
| **Realtime topic**                                            | A Supabase broadcast channel named `channel:<channel_id>` that streams message and reaction changes to subscribed clients.                                                                          |
| **RLS**                                                       | Row-level security. Postgres policies that decide which rows a user can read/write. The primary authorization mechanism. See [Security](../security.md).                                            |
| **`security definer` function**                               | A Postgres function that runs with the definer's privileges, used for org/membership helpers and privileged operations (`create_invite`, `accept_invite`, `search_messages`).                       |
| **`current_org_id()` / `is_admin()` / `is_channel_member()`** | The three RLS helper functions every policy is written against.                                                                                                                                     |
| **Invite**                                                    | A row in `invites` with a hashed token (`token_hash`) and 7-day expiry; accepted via `accept_invite`.                                                                                               |
| **Publishable / anon key**                                    | The public Supabase key the browser uses. Never the service-role key.                                                                                                                               |
| **`@supabase/ssr`**                                           | The Supabase library that manages cookie-based sessions across Next.js server and client components.                                                                                                |
| **Browser / server / middleware client**                      | The three Supabase client factories in `src/lib/supabase/` for client components, server components/route handlers, and session refresh respectively.                                               |
| **ChatWorkspace**                                             | The main client component (`src/features/chat/chat-workspace.tsx`) holding all chat state and realtime wiring.                                                                                      |
| **chat-parts**                                                | Presentational subcomponents (`src/features/chat/chat-parts.tsx`) consumed by `ChatWorkspace`.                                                                                                      |
| **`tsvector` / `websearch_to_tsquery`**                       | Postgres full-text search primitives backing `search_messages`.                                                                                                                                     |
| **Tech-debt marker**                                          | A `TODO`/`FIXME`/`HACK`/`XXX` comment; the `scan-tech-debt` script requires each to reference a tracked issue. See [Tooling](../how-to-contribute/tooling.md).                                      |
